根目录通过 `pnpm-workspace.yaml` 声明 `apps/*` 与 `packages/*` 两个子包集合，`package.json` 中的 scripts 以 `--workspace=` 为契约串联依赖：先 `build:shared` 编译共享库，再并行启动 `dev:server` 与 `dev:web`（由 concurrently 管理），并提供 `build:server`、`build:web`、`start` 等面向部署的顶层命令；所有子模块共享同一份 Node 引擎版本约束（engines.node >= 20）与 .tool-versions 指定的工具链，形成“共享库 → 服务端/前端应用”的单向依赖树。

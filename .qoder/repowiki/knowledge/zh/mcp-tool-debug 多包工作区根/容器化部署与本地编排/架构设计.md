本模块是纯声明式部署层，不包含业务代码，围绕三个核心工件组织：
- `Dockerfile` 使用多阶段构建（build → api → web）：第一阶段以 node:22-alpine 安装 musl 编译依赖并执行 `npm run build:server` 与 `build:web`，第二阶段仅复制产物到最小运行时，API 通过 dumb-init 作为 PID 1 启动 Node 进程，Web 由 nginx:1.27-alpine 托管静态资源。
- `docker-compose.yaml` 定义 `api` 与 `web` 两个服务，通过命名卷 `mcp-tool-debug-data` 持久化 SQLite 文件，`web` 依赖 `api` 的健康检查（`/api/health`），端口映射全部通过 `.env` 中的 `WEB_PORT` / `API_PORT` 覆盖。
- `nginx.conf` 将 `/api/*` 反向代理至内部 `api:8787`，其余请求走 SPA 回退 `try_files`，并关闭缓冲与缓存以支持长连接。
- `deploy.sh` 封装 `docker compose --env-file .env -f deployment/docker-compose.yaml` 子命令（up/down/restart/logs/status），自动从 `.env.example` 生成 `.env`。
依赖方向单向：脚本 → compose → Dockerfile → 镜像；compose 通过环境变量注入数据库方言与 CORS 策略，使同一套镜像同时支持 SQLite 与 PostgreSQL。

本仓库未引入专用配置库（如 dotenv、config、conf），而是采用最简方案：通过 Node.js 原生 process.env 直接读取运行时环境变量，配合 deployment/.env.example 提供默认值模板，由 Docker Compose 注入到容器环境中。

### 1. 使用的框架/工具
- Node.js process.env：所有运行时配置均从进程环境变量读取，无中间层封装。
- .env.example：位于 deployment/ 下，作为部署时环境变量的参考模板，不随代码提交实际 .env 文件。
- Docker Compose：在 deployment/docker-compose.yaml 中通过 environment: 字段将宿主机变量映射进容器，并支持 ${VAR:-default} 语法提供默认值。

### 2. 关键文件
- apps/server/src/index.ts — 启动入口，读取 PORT、CORS_ORIGIN。
- apps/server/src/db/client.ts — 数据库连接选择逻辑，读取 DB_DIALECT、DATABASE_URL，并据此动态加载 SQLite 或 PostgreSQL 驱动。
- deployment/.env.example — 环境变量示例与注释说明。
- deployment/docker-compose.yaml — 服务编排与环境变量注入。
- scripts/mock-mcp-server.ts — 开发脚本，读取 MOCK_MCP_PORT、MOCK_MCP_SESSION_MODE、MOCK_MCP_SLOW_DELAY_MS 等调试开关。

### 3. 架构与约定
- 按功能域划分环境变量前缀：服务器端口/跨域使用 PORT、CORS_ORIGIN；数据库相关统一以 DB_*、DATABASE_URL 命名；Mock 脚本使用 MOCK_MCP_* 前缀。
- 类型推断 + 回退默认值：每个变量在使用处就近做 Number() 转换并提供 ?? 默认值，避免集中式配置校验。
- 方言自动推断：当 DB_DIALECT 未显式设置时，根据 DATABASE_URL 前缀 (postgres:// / postgresql://) 自动判定为 postgres，否则回退 sqlite。
- SQLite 路径解析：file: 协议或相对路径会被解析为相对于 server 根目录的绝对路径，确保容器内数据卷挂载正确。
- 测试覆盖：scripts/session-recovery.test.ts 在运行前临时覆写 DB_DIALECT 和 DATABASE_URL，验证不同方言行为。

### 4. 开发者应遵循的规则
- 新增配置项：在 deployment/.env.example 中添加对应条目及中文注释，并在 docker-compose 的 environment: 中使用 ${VAR:-default} 形式暴露。
- 保持就地读取：不要在模块顶层集中声明配置对象，而是在实际使用处直接从 process.env 取值并赋予默认值。
- 敏感信息不入仓：仅提交 .env.example 模板，真实 .env 文件已在 .gitignore 中排除；生产密钥通过 CI/CD 或宿主机的环境变量注入。
- 方言切换：优先通过显式设置 DB_DIALECT=postgres|sqlite 控制，URL 前缀推断仅作为兜底。

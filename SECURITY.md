# Security Policy / 安全策略

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting or open a draft security advisory for this repository. Do not publish credentials, access tokens, private MCP endpoints, or reproducible exploit details in a public issue.

Include the affected version or commit, impact, reproduction conditions, and a minimal proof of concept with all secrets removed.

## 漏洞报告

请优先使用 GitHub 私有漏洞报告或安全公告草稿。不要在公开 Issue 中发布凭据、访问令牌、私有 MCP 地址或可直接利用的攻击细节。

报告中请说明受影响版本或提交、影响范围、复现条件，并提供已经移除全部敏感信息的最小复现。
